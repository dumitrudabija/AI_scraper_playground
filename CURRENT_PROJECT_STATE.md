## 🏗️ **Architecture Overview**

### **Frontend (PWA)**
- **Framework**: React.js
- **Location**: `/pwa/` directory
- **Features**: Responsive design, offline capability, installable
- **Deployment**: Vercel static hosting
- **API Integration**: Connects to `/api/reports/latest` endpoint

### **Backend (API)**
- **Runtime**: Node.js (Vercel Serverless Functions)
- **Location**: `/api/` directory
- **Main Endpoints**:
  - `GET /api/reports/latest` - Fresh AI news data (primary endpoint)
  - `POST /api/scrape/optimized` - Manual scraping trigger (secondary)
- **AI Enhancement**: Anthropic Claude API integration
- **Fallback System**: Real URLs when RSS scraping fails

### **Data Processing**
- **Method**: Native Node.js RSS parsing
- **No Dependencies**: No Python, no external packages
- **Real-time**: Fresh data generated on each request
- **Smart Fallback**: Real AI news source URLs instead of example.com
- **AI Summaries**: Short, catchy phrases generated by Claude

## 🔧 **Technical Implementation**

### **RSS Scraping Process**
1. **Parallel Fetching**: All 11 sources scraped simultaneously
2. **XML Parsing**: Custom regex-based RSS parser
3. **Content Cleaning**: HTML tag removal, entity decoding
4. **Quality Filtering**: Skip short, empty, or URL-like titles
5. **Data Aggregation**: Combine, deduplicate, sort by date
6. **AI Enhancement**: Claude generates catchy summaries
7. **Response Generation**: JSON format for PWA consumption

### **Error Handling & Fallback**
- **Per-Source Fallback**: Failed sources don't break entire process
- **Smart Fallback Data**: Real AI news source URLs (no example.com)
- **Timeout Protection**: 10-second timeout per RSS feed
- **Graceful Degradation**: App continues working with partial data
- **AI Fallback**: Original descriptions if AI enhancement fails
